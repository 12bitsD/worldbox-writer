# Sprint 21 Plan：前端沉浸式交互重构与 UI 命名重塑

**文档状态**：Done
**版本目标**：Sprint 21 / UX & UI Transformation
**Sprint 周期**：前端体验升维
**定位**：在 Sprint 20 双视窗基础上的“沉浸感”与“隐性交互”升级
**作者**：Agent
**最后更新**：2026-04-24

---

## 1. Sprint 21 要解决什么问题

Sprint 20 成功实现了双视窗（Split-view）和基础的设定快改，但目前的界面仍然带有强烈的“后台管理脚本”味道：
1. 命名偏向程序员思维（如 WorldPanel、Telemetry），增加了普通写手的理解负担。
2. 色彩和排版缺乏“创世”和“文学”的沉浸感。
3. 干预面板（InterventionPanel）在底部占据过大空间，且视觉突兀。
4. 分支面板和关系图谱缺乏空间感和直接操作能力。

Sprint 21 的目标是通过“减少显性控件，增加隐性功能”，将界面从“数据监控台”彻底升级为类似 Notion/Linear 的“沉浸式创世引擎”。

---

## 2. Sprint Goal

**通过通俗的 UI 命名、幽灵命令行、优雅的配色体系以及可视化的分支与图谱，打造专业级 SaaS 体验。**

Sprint 21 结束时，项目具备：

- 全局统一的通俗中文 UI 命名（设定集、故事正文、推演日志等）
- 极简、文学感、无边界的全新色彩体系（羊皮纸白、造物紫等）
- 故事正文视窗全屏化主导，推演日志降级为辅助
- 底部突兀的干预表单升级为内联“幽灵命令行 (Ghost Command Line)”
- 运行状态 (Status) 精简为底部呼吸灯胶囊
- 时间线 (Timelines) 升级为地铁图/Git Graph 可视化树状图
- 关系网 (Network) 支持直接拖拽连线修改人物关系

---

## 3. 方案骨架

### 3.1 承诺交付

1. **Phase 1: 净化与重命名 (已完成)**
   - 全局替换 UI 文案：`WorldPanel` -> 设定集，`StoryFeed` -> 故事正文/推演日志，`InterventionPanel` -> 剧情引导，`BranchPanel` -> 时间线，`RelationshipPanel` -> 关系网。
   - 更新 `index.css`，应用 `#FDFBF7` (暖纸白) 和 `#8B5CF6` (造物紫) 的新配色。
   - 调整 `StoryFeed` 布局比例，使阅读区占据主导。
   - 重构 `InterventionPanel`，实现极简的毛玻璃幽灵命令行输入。

2. **Phase 2: 融合与行内交互**
   - 优化推演日志的展示，使其更符合溯源辅助工具的定位。
   - 完善划词菜单与行内设定编辑的交互流畅度。

3. **Phase 3: 空间化与可操作图谱**
   - 引入可视化时间线组件（取代目前的文字分支列表）。
   - 升级 `RelationshipPanel`，支持拖拽连线（基于已有的 API）。

### 3.2 非目标

- 不在本次 Sprint 改变后端的推演双循环架构。
- 不做极其复杂的多人协作实时光标同步。

## 4. 方案收敛记录

### Round 1：原有的干预弹窗 (Drawer)
问题：
- 用户需要点击干预按钮，再在右侧弹窗中输入指令，路径过长且遮挡阅读区。
结论：否决。改为底部悬浮的“幽灵命令行”，所见即所得。

### Round 2：Telemetry 滚动日志
问题：
- 右侧大面积的滚动日志对普通创作者来说是视觉噪音。
结论：降级。将其精简为“工程呼吸灯 (Status Chips)”并放在左侧/底部，释放右侧空间给“时间线”和“关系网”。
