# 质量评估框架（评测协议与 LLM-as-judge）

本文件聚焦**如何做评测**。网文三轴定义、毒点红线详见 [WEB_NOVEL_CRITERIA.md](./WEB_NOVEL_CRITERIA.md)；四档阈值、北极星详见 [orchestrator/README.md](../orchestrator/README.md)。

---

## 评测协议

每档进入下一档前，指标必须达标且至少跑 2 轮稳定不回退。

1. **样本量**：L1 跑 5 篇短篇；L2 跑 3 篇中篇；L3 跑 1 部长篇 + 短篇校准；L4 长篇 + 多风格短篇
2. **盲评**：至少 3 名评委独立打分，去掉最高最低取均值
3. **对标盲测**：同 prompt 由对标对象生成，A/B 随机顺序，评委选择更"爽/期待感更强/更流畅"的一篇，得出胜率
4. **客观指标**：每篇跑文本分析脚本，产出有效剧情密度、断章悬念率、毒点命中率
5. **自动迭代阶段用 LLM-as-judge**；跨档前必须用真人评委校准一次，防止 LLM 评委自我循环偏差

## LLM-as-judge Prompt 模板 (网文商业化版)

```text
你是一位资深中国网络文学（网文）主编，同时也是极其挑剔的老书虫。请对以下小说片段进行商业化网文视角的评估。

评分维度（每项 1-10，5 分=及格，7 分=好，9 分=优秀，10 分=神作）：

【一、情绪与爽点轴】
1. 期待感建立：是否提前给出了具体的“利益可视化”（读者知道主角要拿到什么）并设定了明确的危机倒计时？
2. 爽点爆发：主角获得收益的瞬间，是否在 500 字内出现了配角/敌人的剧烈反馈（震惊/恐惧）？是纯武力碾压，还是高级的“规则重塑”？
3. 抑扬节奏：吃瘪与反击的比例是否健康？压抑比是否足够大（敌人的嚣张远大于主角的表面实力）？

【二、网文结构轴】
4. 黄金开局/断章艺术：如果是开篇，是否在 1500 字内立住了金手指和核心生存危机？如果是章末，是否完美卡在了“动作中断”、“悬念揭晓”或“利益结算”的瞬间？
5. 信息给配：世界观设定是否自然融入剧情互动中？（大段旁白说明文强行科普直接低分）

【三、商业文笔轴】
6. 阅读顺滑度：句子是否短平快？是否没有生僻词和复杂长句，能一目十行？
7. 画面感/张力：战斗和场景是否有画面感（招式、距离、速度、破坏力）？
8. 对话网感：对话是否符合网文语境？（翻译腔、莎士比亚腔、书面语说教直接低分）

【四、神作进阶轴 (God-Tier) - 仅在评测 L3/L4 级文本时启用】
9. 深度与伏笔：主线是否探讨了高于个人恩怨的母题？当前情节是否巧妙回收了极早前的草蛇灰线？
10. 反派智商与人格：反派动机是否具备逻辑正当性？信息博弈与战术选择是否与主角对等（而非沦为垫脚石）？
11. 抉择与代价：主角是否面临了没有完美解的电车难题？是否为了底线/胜利支付了不可逆的惨痛代价？
12. 规则博弈：战斗是否通过信息差、环境和能力克制取胜，而非单纯的数值/能量对轰？

【五、毒点与 AI 味红线（一票否决，存在即视为 0 分）】
13. 强行降智：反派明知主角有底牌依然贴脸嘲讽、反派死于话多不补刀、高智商人物轻信情报、主角犯“圣母癌”放走死敌。
14. 战力/设定崩坏：主角无代价跨越极大境界杀敌（数值无脑溢出），或前文规则被毫无解释地打破（机械降神），蓝条无限。
15. 说教味/爹味：结尾强行总结升华，输出价值观，主角爱给人当导师。
16. 典型 AI 幻觉：过度比喻（宛如...仿佛...）、三连排比泛滥、翻译腔（哦，我的上帝）。

输出 JSON：
{
  "scores": {
    "anticipation": X, "catharsis": X, "pacing": X,
    "hook_or_cliffhanger": X, "info_pacing": X,
    "readability": X, "visual_action": X, "dialogue_webness": X
  },
  "god_tier_scores": {
    "foreshadowing_depth": X, "antagonist_iq": X, 
    "moral_dilemma_and_cost": X, "rule_based_combat": X
  },
  "toxic_flags": {
    "forced_stupidity": false, "power_collapse": false, 
    "preachiness": false, "ai_hallucination": false
  },
  "overall": X,
  "critical_issues": ["最严重的问题，如命中毒点请详细指出"],
  "best_line": "最抓人/最爽的一句",
  "worst_line": "最毒/最出戏的一句",
  "one_line_suggestion": "一句话修改建议（例如：砍掉前三段的设定科普，直接从打斗开始）"
}
```

## 毒点检测与拦截标准 (一票否决)

| 毒点 | 定义 | 判定标准 | 拦截动作 |
| ------ | --------------- | ------------------- | -- |
| 强行降智 | 为了推进剧情让反派/主角变成弱智 | 行为逻辑无法自洽 | **阻断生成，发回重写** |
| 说教味 | 结尾总结升华，输出价值观 | 出现“这让他明白了...”、“不仅是...更是...” | **阻断生成，发回重写** |
| 设定说明文 | 大段旁白直接交代世界观 | 连续 >100 字无互动的纯设定描写 | 扣分，强制修改 |
| AI 幻觉修辞 | 比喻/排比泛滥，翻译腔 | 比喻密度 > 3 次/千字 | 扣分，强制修改 |

---

## 迭代停机规则

每轮迭代必须在“网文三轴”（情绪爽点、结构、商业文笔）中至少一轴产生可量化提升，且**绝不能命中任何一条毒点红线**。
连续 3 轮无提升或反复命中同一毒点 → **停止迭代，重新评估方向。**
