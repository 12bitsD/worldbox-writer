# 质量评估框架（评测协议与 LLM-as-judge）

本文件聚焦**如何做评测**。双轴定义、四档阈值、北极星详见 [orchestrator/README.md](../orchestrator/README.md)。

---

## 评测协议

每档进入下一档前，指标必须达标且至少跑 2 轮稳定不回退。

1. **样本量**：L1 跑 5 篇短篇；L2 跑 3 篇中篇；L3 跑 1 部长篇 + 短篇校准；L4 长篇 + 多风格短篇
2. **盲评**：至少 3 名评委独立打分，去掉最高最低取均值
3. **对标盲测**：同 prompt 由对标对象生成，A/B 随机顺序，评委选择更"精彩 / 文笔更好"的一篇，得出胜率
4. **客观指标**：每篇跑文本分析脚本，产出句长方差、TTR（词汇丰富度）、陈词滥调命中率
5. **自动迭代阶段用 LLM-as-judge**；跨档前必须用真人评委校准一次，防止 LLM 评委自我循环偏差

## LLM-as-judge Prompt 模板

```
你是一位资深中文小说编辑，同时也是严厉的质量把关人。请对以下小说片段进行评估。

评分维度（每项 1-10，5 分=及格，7 分=好，9 分=优秀）：

【故事力】
1. 钩子：前 500 字是否让你想继续读？
2. 冲突密度：每千字有几个核心冲突/张力点？
3. 人物动机一致性：人物决策是否可被倒推解释？
4. 悬念维持：读完后想不想看下一章？

【文笔】
5. 句式多样性
6. 意象新鲜度
7. 对话辨识度
8. 信息密度
9. 可读性

【AI 味检测（重点）】
10. 过度比喻（1=泛滥，10=克制）
11. 过度排比（1=泛滥，10=克制）
12. 段落连贯性（1=严重断裂，10=完全连贯）
13. AI 味综合（1=明显 AI，10=完全像人）

输出 JSON：
{
  "scores": {
    "hook": X, "conflict_density": X, "character_motivation": X, "suspense": X,
    "sentence_variety": X, "imagery_freshness": X, "dialogue_distinctiveness": X,
    "information_density": X, "readability": X,
    "metaphor_abuse": X, "parallelism_abuse": X, "coherence": X, "ai_flavor": X
  },
  "overall": X,
  "critical_issues": ["最严重的问题"],
  "best_line": "最好的一句",
  "worst_line": "最差的一句",
  "one_line_suggestion": "一句话改进建议"
}
```

## AI 写作常见问题检测标准

| 问题     | 定义              | 判定标准                | 权重 |
| ------ | --------------- | ------------------- | -- |
| 过度比喻   | 每句都要比喻，像在炫技     | 比喻密度 > 3 次/千字 = 扣分  | 高  |
| 过度排比   | 三连排比成瘾          | 排比结构 > 2 次/千字 = 扣分  | 高  |
| 段落断裂   | 段落间逻辑突然跳转       | 前后段无因果/时间连接 = 扣分    | 高  |
| 陈词滥调   | "宛如…般"、"仿佛…似的"  | cliché 命中率 > 5% = 扣分 | 中  |
| 注水废句   | 无信息量的句子         | 重复/空洞/废话 = 扣分       | 中  |

---

## L1 通过阈值

| 维度               | 最低分 | 说明        |
| ---------------- | --- | --------- |
| 过度比喻             | ≥ 6 | 比喻克制精准    |
| 过度排比             | ≥ 6 | 排比克制      |
| 段落连贯性            | ≥ 7 | 必须连贯      |
| 可读性              | ≥ 7 | 必须读得下去    |
| AI 味综合           | ≥ 6 | 不能太 AI    |

---

## 迭代停机规则

每轮迭代必须在双轴（故事力 + 文笔）中至少一轴产生可量化提升，否则计入"连续无提升"。**连续 3 轮无提升 → 停止迭代，重新评估方向。**
