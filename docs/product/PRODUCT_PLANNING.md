# WorldBox Writer: 产品演进规划 (Product Planning)

**文档状态**：Active (v0.5.0+)
**作者**：Manus AI

本文档基于 WorldBox Writer 的产品愿景，自顶向下（Top-Down）对未来版本进行规划。结构为：**Product Vision → Release Goals (阶段大目标) → Epics (史诗故事) → User Stories (用户故事) → Sprints (冲刺排期)**。

---

## 1. 产品愿景 (Product Vision)

**"打造一个让创作者以'创世神'视角，通过干预 Agent 沙盒世界来生成长篇小说的生产力工具。"**

WorldBox Writer 的核心差异化在于**从"文本编辑器"升维到"世界引擎"**。我们不追求让 AI 一次性吐出 10 万字，而是让多个 Agent（角色、导演、守卫）在一个有规则的虚拟沙盒中互动，用户通过设定规则和关键节点的干预来引导故事走向，从而解决长篇一致性差、缺乏涌现性的痛点。

---

## 2. 阶段性大目标 (Release Goals)

目前我们已完成 **Release 1 (v0.5.0)**：端到端 MVP，证明了"沙盒推演 + 关键点干预"的技术可行性，并实现了基本的实时流与持久化。

接下来，我们将产品演进划分为三个核心 Release 阶段：

### Release 2: 可见且可控的世界 (v0.6.0 - v0.7.0)
**大目标**：让黑盒的推演过程变得直观可见，让单向线性的故事支持多分支探索，并提供专业的交付能力。
- **价值**：解决当前"沙盒感不足"的问题，让用户真正感受到世界在运转；解决"不能反悔"的痛点，提供分支回溯能力。

### Release 3: 深度创作工作流 (v0.8.0 - v0.9.0)
**大目标**：从"玩具"升级为"工具"，支持用户导入已有大纲，提供富文本润色环境，以及更细粒度的节奏控制。
- **价值**：满足真实作家的生产力需求，降低使用门槛，提升最终文本的文学质量。

### Release 4: 生产就绪与商业化基座 (v1.0.0)
**大目标**：千万字级别的长篇稳定性，多模型智能路由（降本增效），异步高并发支持。
- **价值**：达到可作为 SaaS 产品对外提供服务的技术标准。

---

## 3. 史诗拆解与用户故事 (Epics & User Stories)

以下重点拆解 **Release 2** 和 **Release 3** 的 Epics 和 User Stories。

### Epic 05: 可视化沙盒面板 (Visual Sandbox Dashboard) - *进行中*
*（继承自 MVP，进一步深化表现层）*

- **US-05.02: 动态人物关系图谱 (P0)**
  - 作为创世神，我希望能查看动态更新的人物关系图谱，以便掌握全局势力变化。
- **US-05.03: 真实 Agent 遥测日志 (P2)**
  - 作为创世神，我希望能看到各 Agent 的内部思考过程（如提议被拒绝的原因），以便了解世界运转的底层逻辑。

### Epic 06: 多世界线与分支管理 (Branching Narratives) - *New*
*（WorldBox Writer 的核心杀手锏）*

- **US-06.01: 时间线回溯与分叉 (P0)**
  - 作为创世神，我希望能在任意历史节点"存档"并引入新的干预指令，从而衍生出平行世界线，以便探索不同的故事走向（如"如果主角在这里选择背叛会怎样？"）。
- **US-06.02: 多分支对比面板 (P1)**
  - 作为创世神，我希望能同时推演并对比 2-3 条分支大纲，以便选择最满意的一条进行精细渲染。
- **US-06.03: 故事线版本控制 (P2)**
  - 作为创世神，我希望能像使用 Git 一样管理故事线（commit, checkout, merge），以便安全地进行实验性推演。

### Epic 07: 深度世界构建与资产管理 (Deep Worldbuilding) - *New*
*（将设定的控制权交还给人类）*

- **US-07.01: 大纲与预设导入 (P1)**
  - 作为世界构建师，我希望能上传现有的世界观设定或故事大纲 JSON，以便让系统基于我的存稿进行推演，而不是从零生成。
- **US-07.02: 交互式设定集 Wiki (P1)**
  - 作为世界构建师，我希望能在一个类似 Wiki 的界面中增删改查势力、地点和角色设定，修改后自动触发一致性校验。
- **US-07.03: 多格式专业导出 (P1)**
  - 作为创作者，我希望能将推演完成的小说导出为排版精美的 PDF 和 Word 格式，以便直接交付或继续排版。

### Epic 08: 工作流与生产力增强 (Workflow & Productivity) - *New*

- **US-08.01: 叙事节奏控制器 (P2)**
  - 作为导演，我希望能设置当前阶段的"基调"（如：平缓日常、紧张战斗），以便 GateKeeper 能据此调整 Agent 提议的通过率。
- **US-08.02: 前端富文本编辑器集成 (P2)**
  - 作为叙述者，我希望能在 Narrator 渲染完成后，在一个所见即所得的编辑器中手动润色文本，并将关键修改反向同步给底层记忆系统。

## 4. 冲刺排期 (Sprint Roadmap)

基于上述拆解，我们规划未来 4 个 Sprint 的路线图：

| Sprint | 版本目标 | 核心 Epic | 包含的 User Stories | 预期交付物 |
|---|---|---|---|---|
| **Sprint 6** | v0.6.0 | E05, E07 | US-05.02 (关系图谱)<br>US-07.03 (PDF/Word导出) | 前端动态力导向图谱；<br>后端 PDF/Docx 导出端点。 |
| **Sprint 7** | v0.7.0 | E06 | US-06.01 (时间线分叉)<br>US-06.02 (多分支对比) | 底层 DAG 支持树状分叉；<br>前端新增 Timeline 组件。 |
| **Sprint 8** | v0.8.0 | E07 | US-07.01 (大纲导入)<br>US-07.02 (交互式 Wiki) | 启动时支持 JSON 注入；<br>设定面板升级为全功能 Wiki。 |
| **Sprint 9** | v0.9.0 | E08 | US-08.01 (节奏控制)<br>US-08.02 (富文本反向同步) | 引入 Pacing 参数；<br>前端集成 TipTap/Quill 编辑器。 |

> **注**：Sprint 6 是承上启下的关键，它将通过"关系图谱"彻底激活产品的"沙盒感"，同时通过"专业导出"闭环创作者的最基本交付需求。
